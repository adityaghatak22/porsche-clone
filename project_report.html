Abstract

The Porsche Premium Automotive Web Application is a comprehensive, full-stack mini-project developed as part of the Web Programming Laboratory curriculum for Semester IV. The application replicates the core digital experience of a luxury automotive brand's official website, incorporating user authentication, real-time car configuration, model browsing with price-based filtering, and an integrated test-drive booking system.

The back-end is powered by PHP 8 using PDO (PHP Data Objects) for secure, prepared-statement-driven interaction with a MySQL relational database. The front-end employs pure HTML5 semantic markup, vanilla CSS3 (with custom properties and animations), and vanilla JavaScript for AJAX-based dynamic content updates — showcasing that premium UI/UX can be achieved without heavy front-end frameworks.

Key features include: a cinematic hero showcase section with scroll-based parallax, a real-time car configurator with exterior colour swatches, a CRUD-enabled test-drive management module, Google Maps integration, a smart international phone-number validator, an auto-hiding navigation bar, and full password/email security validation on all authentication forms.

The application successfully demonstrates a production-grade MVC-like file structure, responsive design, session-based access control, and SQL-injection prevention — making it a robust demonstration of modern PHP web development practices.

Keywords

PHP, MySQL, PDO, Prepared Statements, AJAX, HTML5, CSS3, Vanilla JavaScript, Session Management, Responsive Web Design, CRUD, Authentication, Automobile, Configurator, Google Maps API, International Phone Validation.

Introduction

In today's digital-first world, every major automotive brand invests in interactive online experiences that allow customers to explore vehicles, configure their ideal model, and book services — all without visiting a dealership. Our project is inspired by this real-world use case, taking the globally recognised Porsche brand as a model and building a functionally equivalent web application from the ground up.

The project was conceived to bridge the gap between academic PHP fundamentals and industry-level web development. Rather than building a generic CRUD application, we designed an experience that is aesthetically compelling, architecturally sound, and feature-rich. The application is served locally via XAMPP (Apache + MySQL) and is structured to be deployable on any standard LAMP/WAMP stack with minimal configuration.

Scope of the Project

The project covers the following functional domains:

  • User Management: Secure registration with password-complexity enforcement, email-format validation, session-based login and logout.
  • Model Showcase: Dynamic car listing from a MySQL database, AJAX-powered search with keyword and price-range filters.
  • Car Configurator: Real-time exterior colour selection with live car image updates; configuration save and retrieval ("My Builds").
  • Test Drive Booking: Full CRUD for test drive appointments with model selection, international phone validation, location map, and status management.
  • UI/UX: Cinematic hero sections, auto-hiding navbar, glassmorphism-inspired dark theme, Bebas Neue/Barlow typography, micro-animations.
Background

Traditional academic PHP projects often focus narrowly on database connectivity. This project deliberately integrates the full stack — from SQL schema design to pixel-perfect CSS — giving all four team members exposure to every layer of a production web application. The project also reinforces software engineering practices such as input sanitisation, error handling, code modularity (PHP includes), and semantic HTML.

Problem Statement & Objectives
Problem Statement

Automotive companies require sophisticated digital platforms that allow end-users to explore and interact with their products online. Most academic-level web projects, however, are limited to simple login/register systems or basic CRUD boards. There is a clear gap between what students learn in lab exercises and what a real-world commercial web application entails in terms of security, UX, performance, and feature completeness.

This project addresses that gap by building a fully functional, multi-page PHP/MySQL web application that simulates the core offerings of a luxury automotive brand's digital portal.

Objectives
  1. Design and implement a secure authentication system with server-side and client-side validation for both email format and password complexity (uppercase, number, symbol).
  2. Build a dynamic model listing page that uses AJAX to filter cars by search term and price bracket without full page reloads.
  3. Develop a real-time car configurator that changes the displayed vehicle image based on the user's colour selection and persists the chosen configuration to the database.
  4. Implement a test drive booking system with complete create / read / update / delete operations, international phone-number validation by country code, and Google Maps integration.
  5. Create a visually premium, responsive UI with a dark colour palette, professional typography, smooth animations, and an auto-hiding navigation bar.
  6. Ensure data security through PDO prepared statements on every SQL query, HTML output escaping, and session-based access control.
Expected Outcomes
  • A fully navigable multi-page web application running on a local LAMP/WAMP stack.
  • Complete user registration, login, session management, and logout flows.
  • Dynamic, database-driven car listings with real-time search and price filtering.
  • A working car configurator persisting user selections to the database.
  • A test-drive booking portal with full CRUD and map integration.

Data flows from the browser to PHP scripts via HTML forms (POST) or AJAX requests (GET), processed server-side, and returned as rendered HTML or JSON.

Algorithm & Implementation Details
1. Password Validation Algorithm (register.php & login.php)

Server-side validation uses PHP's preg_match with four discrete checks:

  • Length ≥ 8 characters
  • At least one uppercase letter: /[A-Z]/
  • At least one digit: /[0-9]/
  • At least one special character: /[\W_]/

Passwords are stored using PHP's password_hash($pwd, PASSWORD_BCRYPT) and verified with password_verify(). Email format is validated using filter_var($email, FILTER_VALIDATE_EMAIL).

2. AJAX Search + Price Filter (ajax_search_models.php)

The models page sends a GET request with two parameters — q (keyword) and p (price range). The server builds a dynamic SQL query with WHERE 1=1 as a base:

  • If q is non-empty: appends AND model_name LIKE :q (prepared parameter).
  • If p = "under80": appends AND price < 80000.
  • If p = "80to120": appends AND price BETWEEN 80000 AND 120000.
  • If p = "over120": appends AND price > 120000.
  • Always appended: ORDER BY price ASC.

The result is JSON-encoded and consumed by JavaScript to re-render the grid without a page reload.

3. International Phone Validation (JavaScript)

A country-code map stores the expected subscriber-digit counts for major nations. On every input event:

  1. Check that the value starts with +.
  2. Sort known country codes longest-first to avoid prefix collisions (e.g. +971 before +9).
  3. Match the entered prefix against the map and compute the subscriber digits.
  4. Call setCustomValidity() with a descriptive message if the count is wrong; clear it if correct.
  5. Unknown prefixes fall back to the ITU range of 7–15 total digits.
4. Auto-Hiding Navbar (includes/header.php)

A scroll event listener tracks lastScrollTop. If the user scrolls down more than 100 px, the CSS class navbar-hidden is added (CSS transform: translateY(-100%)). Scrolling up removes the class. A second class scrolled adds a semi-transparent background after 50 px of downward scroll, keeping the navbar readable over all content.

Conclusion & Future Scope
Conclusion

The Porsche Premium Automotive Web Application successfully demonstrates the integration of modern PHP back-end development with a polished, premium front-end design. All project objectives were achieved:

  • Security: Bcrypt password hashing, PDO prepared statements, HTML output escaping, and session-based access control protect the application against common web vulnerabilities (XSS, SQL injection, session hijacking).
  • Functionality: Full CRUD for both car configurations and test-drive bookings is implemented with clean user feedback and data persistence.
  • User Experience: The dark, cinematic design language — combined with micro-animations, an auto-hiding navbar, real-time AJAX filtering, and thoughtful form validation — delivers a premium feel rarely seen in academic projects.
  • Code Quality: PHP includes for shared components (header/footer), a single PDO connection file (db.php), and consistent CSS class naming conventions ensure the codebase is maintainable and extensible.

The project gave all four team members hands-on experience with the complete web development lifecycle — from database design and back-end routing to front-end UI polish and client-side scripting. It reinforces that building a professional-grade web application requires equal attention to aesthetics, security, and architecture.

Future Scope
  • Payment Integration: Integrate a sandbox payment gateway (Razorpay / Stripe) to simulate vehicle reservation deposits.
  • Admin Dashboard: A separate admin portal to manage all users, bookings, and car inventory with charts and analytics.
  • REST API: Expose the back-end as a JSON REST API to support a future React/Vue.js front-end.
  • Email Notifications: Use PHPMailer to send booking confirmation and reminder emails.
  • Progressive Web App (PWA): Add a service worker and manifest to enable offline browsing and home-screen installation on mobile devices.
  • Multi-language Support: Internationalise the UI (English / Hindi / German) to broaden accessibility.
  • Car Comparison Tool: Allow users to select two models side-by-side and compare specifications in a table.
